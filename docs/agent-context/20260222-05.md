# [TICKET-20260222-05] `gameStorage` の `origin/value` 整合バリデーション強化

## 目的 / 対象読者（coding agent）
保存盤面データの不整合（例: `origin="given"` かつ `value=null`）を読み込み時に排除し、UI状態の一貫性を担保する。主対象は盤面ストレージと状態復元を実装するエージェント。

## 作成日
2026-02-22

## 優先度 / 状態
- 優先度: Low-Medium
- status: `TODO`

## 背景と問題
1. 現行 `isValidCell` は `value === null` を無条件許容し、`origin` との整合を検証していない。
- 根拠: `src/lib/gameStorage.ts:26`
- 根拠: `src/lib/gameStorage.ts:30`

2. セル編集可否は `origin` に依存して決まるため、不整合データが復元されるとUI上の意味が崩れる。
- 根拠: `src/components/SudokuCell.tsx:36`
- 根拠: `src/components/SudokuCell.tsx:37`

## 実装方針
1. `isValidCell` に `origin/value` の整合ルールを追加する。
- `origin === "empty"` のときのみ `value === null` を許容。
- `origin === "given"` または `origin === "user"` のときは `value` が整数 `1..9` のみ許容。

2. 不整合セルを含む `board` は無効と判定し、`loadGameState()` は `null` を返す。
- 既存の安全フォールバック（保存データ未使用で初期化）を維持する。

3. テストを拡張する。
- `src/lib/__tests__/gameStorage.test.ts` に以下を追加:
  - `given + null` が読み込み拒否される。
  - `user + null` が読み込み拒否される。
  - `empty + 5` が読み込み拒否される。

## 受け入れ基準
1. `origin/value` 不整合データは読み込み時に拒否される。
2. 正常データ（現行仕様の保存値）は引き続き読み込み可能。
3. `src/lib/__tests__/gameStorage.test.ts` の新規/既存テストがすべて成功する。

## 検証手順
1. `npm run test -- src/lib/__tests__/gameStorage.test.ts`
2. `npm run test`

## 非目標（今回やらないこと）
- 保存データのマイグレーションバージョン更新
- 盤面編集仕様自体の変更

## リスク / ロールバック
- リスク: 既存環境の破損データが読み込めなくなり、初期化扱いになる。
- ロールバック: 整合ルールを段階導入し、無効値を補正変換する方針へ切り替える。
