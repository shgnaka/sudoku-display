# [TICKET-20260222-01] `useKeyboardInset` 回転後誤判定対策

## 目的 / 対象読者（coding agent）
モバイル端末の画面回転後にキーボード非表示でも `inset` が残留する問題を防ぎ、`solve-no-scroll` のレイアウト誤動作を解消する。主対象は Solve 画面と viewport 制御を実装するエージェント。

## 作成日
2026-02-22

## 優先度 / 状態
- 優先度: High
- status: `DONE`

## 背景と問題
1. `baselineRef` が「最大 visibleHeight 固定」になっているため、回転後の新しい基準高さへ追従できず誤差が残る。
- 根拠: `src/lib/useKeyboardInset.ts:35`
- 根拠: `src/lib/useKeyboardInset.ts:36`
- 根拠: `src/lib/useKeyboardInset.ts:38`

2. `orientationchange` で同一ロジックを再実行するだけで、基準値再初期化が行われない。
- 根拠: `src/lib/useKeyboardInset.ts:46`

3. 既存テストに「回転後ベースライン再基準化」の検証がない。
- 根拠: `src/lib/__tests__/useKeyboardInset.test.tsx:74`
- 根拠: `src/lib/__tests__/useKeyboardInset.test.tsx:114`

## 実装方針
1. `useKeyboardInset` の基準値更新戦略を変更する。
- `orientationchange` 発生時は `baselineRef.current` を現在の `visibleHeight` で再初期化する。
- `resize`/`scroll` 時も「明確な再基準化イベント」と判定できる場合はベースラインを更新できるようにする。

2. `updateInset` の判定を2段化する。
- 段1: 基準値更新（回転・再基準化判定）。
- 段2: `rawInset` 算出と `KEYBOARD_THRESHOLD_PX` 適用。

3. テストを追加する。
- `src/lib/__tests__/useKeyboardInset.test.tsx` に「回転後にキーボード非表示状態で inset が 0 に戻る」ケースを追加する。
- 既存ケース（タッチ端末検知、閾値判定、unmount cleanup）は維持する。

## 受け入れ基準
1. 画面回転後、キーボード非表示時の `useKeyboardInset()` が `0` を返す。
2. キーボード表示時の `inset` 算出は既存どおり機能する。
3. `src/lib/__tests__/useKeyboardInset.test.tsx` の全テストが成功する。
4. Solve 画面の `keyboard-spacer` が回転後に過剰な高さを保持しない。

## 検証手順
1. `npm run test -- src/lib/__tests__/useKeyboardInset.test.tsx`
2. `npm run test`

## 非目標（今回やらないこと）

- Solve 画面レイアウト構成の再設計

## リスク / ロールバック
- リスク: 再基準化条件を広げすぎると通常入力中に `inset` が揺れる可能性がある。
- ロールバック: `useKeyboardInset` の変更をチケット単位で戻し、テスト追加分のみ維持して再設計する。
