# Agent Context Docs

このディレクトリは、改善案件の背景事情をコーディングエージェントへ共有するための永続ドキュメント置き場です。

## 最上位ルール（必須）
1. 改善案件を記録する前に、まず `docs/agent-context/` 配下を確認する。
2. 類似案件のファイルがある場合は、新規作成せず既存ファイルへ追記する。
3. 追記時は `更新日（YYYY-MM-DD）` を明記する。
4. 類似案件がない場合のみ、`TEMPLATE.md` を使って新規作成する。

## 運用ルール
- ファイル命名規則: `<scope>-<topic>-audit.md`
- 日付は絶対日付で記載: `YYYY-MM-DD`
- `改善余地` には必ず `ファイル:行番号` を含める
- 実装順序を `次に実装すべき順序` で明示する

## 作成手順
1. `docs/agent-context/` 配下を確認して類似ファイルの有無を判断する。
2. 類似がなければ `docs/agent-context/TEMPLATE.md` を複製して新規作成する。
3. `目的 / 対象読者`、`調査日`、`現状要約` を埋める。
4. `改善余地` を優先度付きで記載し、各項目に `ファイル:行番号` を付与する。
5. `非目標` と `次に実装すべき順序` を記載して完了とする。

## 既存ファイル更新時のルール
- 既存判断を更新する場合は、変更理由を明記する。
- 追記後に章構成が崩れていないか確認する。
